WellHeard AI API Documentation

Welcome to WellHeard AI, a WebSocket-based voice AI API for telephony partners. Connect your phone system, stream audio bidirectionally, and let our AI handle conversations — from insurance SDR calls to healthcare intake, collections, and more.

Key Features

  • Real-time bidirectional audio — Stream mulaw-encoded audio both ways
  • Intelligent call handling — Voicemail detection, warm transfers, and more
  • Flexible data integration — Pull lead data from your systems on demand
  • Merge fields — Personalize AI responses with dynamic customer data
  • Webhook notifications — Receive call results and transcripts post-call

Base URLs

WebSocket: wss://wellheard.ai/v1/partner/audio
Dashboard: https://wellheard.ai/v1/agent/dashboard

Authentication

WellHeard AI uses API keys for authentication. Pass your API key in the WebSocket connection URL as a query parameter or in the start event.

Query Parameter

URL 
wss://wellheard.ai/v1/partner/audio?api_key=YOUR_API_KEY
Security: Always use wss:// (WebSocket Secure). Never transmit API keys over unencrypted connections. Treat your API key like a password — do not commit it to version control.

You can find your API key in your WellHeard dashboard.

WebSocket Connection

The WellHeard API uses WebSocket for low-latency, bidirectional communication with strict audio specifications.

Connection Specifications

Property Value Description
Protocol wss:// WebSocket Secure (encrypted)
Audio Codec mulaw G.711 μ-law compression
Sample Rate 8000 Hz Standard telephony rate
Channels mono Single audio channel
Frame Size 160 bytes Represents 20ms of audio (~50 frames/sec)
Message Order JSON first Send start event as JSON, then binary audio
Audio Format: Raw mulaw frames (160 bytes each). After the initial JSON start event, all audio is sent as binary data. JSON events (transfer, end, etc.) may be interspersed with audio.

Step 1: Connect & Send Start Event

Begin the call by opening a WebSocket connection and immediately sending a JSON start event. This event contains all lead information, configuration, and callback URLs.

Start Event

JSON 
{
  "event": "start",
  "call_id": "your-unique-call-id",
  "direction": "outbound",
  "phone_number": "+16135551234",
  "first_name": "John",
  "last_name": "Smith",
  "full_name": "John Smith",
  "email": "john@example.com",
  "birth_date": "1955-03-15",
  "state": "Florida",
  "city": "Tampa",
  "zip_code": "33601",
  "data_url": "https://your-api.com/calls/{call_id}/data",
  "data_api_key": "your-api-key",
  "webhook_url": "https://your-api.com/calls/webhook",
  "codec": "mulaw",
  "sample_rate": 8000
}

Start Event Field Reference

Field Required Type Description
event Yes string Must be "start"
call_id Yes string Your unique call identifier. Used in data_url and returned in webhook
direction Yes string outbound or inbound
phone_number Yes string E.164 format (e.g., +16135551234)
first_name No string Prospect's first name. Available as merge field
last_name No string Prospect's last name. Available as merge field
full_name No string Full name. Available as merge field
email No string Email address. Available as merge field
birth_date No string ISO 8601 format (YYYY-MM-DD). Available as merge field
state No string US state. Available as merge field
city No string City name. Available as merge field
zip_code No string Postal code. Available as merge field
data_url No string URL to fetch additional lead data. WellHeard will make a GET request
data_api_key No string Auth header sent as Bearer {data_api_key}
webhook_url No string URL to POST call results when the call ends
codec No string Audio codec. Default: mulaw
sample_rate No number Sample rate in Hz. Default: 8000
Custom Fields: Any additional key-value pairs in the start event are accepted as custom fields and become available as merge variables in your AI script.

Step 2: Data Fetch (Pull-Based)

If you provided a data_url in the start event, WellHeard will immediately make a GET request to fetch additional lead data before starting the audio stream.

Data Fetch Request

HTTP 
GET https://your-api.com/calls/{call_id}/data
Authorization: Bearer {data_api_key}

Data Fetch Response

Your endpoint should return a JSON object with any key-value pairs. All fields are merged with the start event fields and become available as merge variables for your AI script.

JSON 
{
  "insurance_type": "health",
  "coverage_amount": "$50,000",
  "policy_status": "active",
  "last_premium_paid": "2024-01-15",
  "delinquent_days": 45
}
Merge Behavior: Fields from data_url response are merged with start event fields. If both contain the same key, the data_url value takes precedence. All merged fields are available for use in merge field templates.

Implementation Notes

  • WellHeard waits up to 5 seconds for your data_url response
  • If your endpoint returns a 4xx error, the call proceeds with only start event data
  • Response must be valid JSON object (not an array or primitive)
  • Use {call_id} placeholder in your data_url — WellHeard replaces it with the actual call_id

Step 3: Audio Streaming

After receiving the ready event from WellHeard, begin streaming audio bidirectionally. Raw mulaw frames are sent both ways with no JSON wrapping.

Ready Event

When WellHeard is ready to receive audio, it sends a ready event with session metadata:

JSON 
{
  "event": "ready",
  "call_id": "partner-xxx",
  "session_id": "partner-xxx",
  "setup_ms": 1250.3
}

Audio Stream Flow

You → WellHeard: Raw mulaw frames (160 bytes) at ~50 frames/sec WellHeard → You: Raw mulaw frames (160 bytes) when AI is speaking No JSON wrapping. Binary data only.

Audio Frame Details

Parameter Value Details
Frame Size 160 bytes Exactly 160 bytes per frame
Duration 20 milliseconds 160 samples at 8000 Hz
Frequency ~50 frames/second Send frames at regular 20ms intervals
Format Binary mulaw G.711 μ-law compressed audio
Channels Mono Single channel only
Frame Synchronization: Send frames at consistent 20ms intervals. Use timers or callbacks from your audio device to maintain proper pacing. Irregular timing can cause audio artifacts.

Audio Streaming Example (Conceptual)

Pseudocode 
// On ready event:
startAudioStream()

// Every 20ms, get audio from call:
audioChunk = phoneDevice.getAudio(160)  // 160 bytes of mulaw
ws.send(audioChunk)                     // Send as binary

// When receiving from WebSocket:
ws.on('message', (data) => {
  if (data instanceof ArrayBuffer) {
    // Raw audio frame from WellHeard AI
    phoneDevice.playAudio(data)
  } else {
    // JSON event
    handleEvent(JSON.parse(data))
  }
})

Step 4: Call Events

During and after the call, WellHeard sends real-time event notifications. You can also send events to WellHeard to control the call.

Events from WellHeard

voicemail_detected

Sent when WellHeard detects voicemail (continuous audio >5 seconds at call start).

JSON 
{
  "event": "voicemail_detected"
}

transfer

Sent when the AI decides to transfer the call to a human agent (warm transfer).

JSON 
{
  "event": "transfer",
  "target": "+19048404634"
}

end

Sent when the AI ends the conversation or the call is terminated.

JSON 
{
  "event": "end",
  "reason": "completed"
}

Events You Can Send to WellHeard

stop

Terminate the call immediately.

JSON 
{
  "event": "stop",
  "reason": "hangup"
}

dtmf

Send a DTMF (Dual-Tone Multi-Frequency) digit to the phone system.

JSON 
{
  "event": "dtmf",
  "digit": "1"
}
DTMF Digits: Valid values are 0-9, *, #, and A-D.

Step 5: Post-Call Webhook

When the call ends, WellHeard POSTs the complete call results and transcript to your webhook URL (if provided in the start event).

Webhook Request

HTTP 
POST {webhook_url}
Content-Type: application/json
Authorization: Bearer {data_api_key}

Webhook Payload

JSON 
{
  "call_id": "your-unique-call-id",
  "wellheard_call_id": "partner-your-unique-call-id",
  "duration_seconds": 127.3,
  "disposition": "transfer_completed",
  "transcript": [
    {
      "role": "agent",
      "text": "Hi John, this is Sarah calling about your insurance coverage..."
    },
    {
      "role": "prospect",
      "text": "Yeah, who's this?"
    }
  ],
  "turns": 8,
  "transfer_initiated": true,
  "voicemail_detected": false
}

Webhook Field Reference

Field Type Description
call_id string Your original call_id from the start event
wellheard_call_id string WellHeard's internal call identifier
duration_seconds number Total call duration in seconds
disposition string Call outcome. See disposition values below
transcript array Array of conversation turns with role (agent/prospect) and text
turns number Number of back-and-forth turns in the conversation
transfer_initiated boolean Whether a transfer to an agent was initiated
voicemail_detected boolean Whether voicemail was detected at call start

Disposition Values

Disposition Description
transfer_completed Call was transferred to a human agent and connected
completed Call completed successfully with AI handling the entire interaction
voicemail Voicemail was detected and AI left a message
no_answer Call went unanswered or timed out
hung_up Prospect hung up during the call
Webhook Retry: WellHeard will retry failed webhooks with exponential backoff up to 3 times. Return a 2xx status code to acknowledge receipt.

Merge Fields

All lead data (from the start event and data_url response) can be used as merge variables in your AI script, greeting, FAQ, and other text fields. Use double curly braces to reference a field.

Merge Field Syntax

Template 
{{field_name}}
{{Field Name}}  // Case-insensitive
{field_name}      // Single braces also work

Examples

Text 
"Hi {{first_name}}, this is Sarah calling about your {{insurance_type}} coverage"
"We see you're in {{state}} with a {{coverage_amount}} policy"
"Your policy expires on {{expiration_date}}"

Field Matching Rules

  • Case-insensitive: {{first_name}}, {{First Name}}, and {{FIRST_NAME}} all resolve to the same field
  • Underscores and spaces: {{first_name}} and {{first name}} are equivalent
  • Custom fields: Any field in your start event or data_url response can be used
  • Missing fields: If a field is not provided, it's replaced with an empty string
  • Escaping: Use {{field_name}} or {field_name} (single braces)

Common Merge Variables

Variable Description
{{first_name}} Prospect's first name
{{last_name}} Prospect's last name
{{full_name}} Full name
{{email}} Email address
{{phone_number}} Phone number
{{state}} State
{{city}} City
{{zip_code}} Postal code
Custom Data: If you include custom fields in the start event or fetch them via data_url (e.g., insurance_type, coverage_amount), they're instantly available as merge variables.

Code Examples

Complete, production-ready examples for connecting to the WellHeard API in different languages.

Node.js / JavaScript

JavaScript 
const WebSocket = require('ws');

const ws = new WebSocket(
  'wss://wellheard.ai/v1/partner/audio?api_key=YOUR_API_KEY'
);

ws.on('open', () => {
  const startEvent = {
    event: 'start',
    call_id: 'call-' + Date.now(),
    direction: 'outbound',
    phone_number: '+16135551234',
    first_name: 'John',
    last_name: 'Smith',
    state: 'Florida',
    data_url: 'https://your-api.com/calls/{call_id}/data',
    webhook_url: 'https://your-api.com/calls/webhook'
  };

  ws.send(JSON.stringify(startEvent));
});

ws.on('message', (data, isBinary) => {
  if (isBinary) {
    // Binary audio frame from WellHeard AI
    playAudioToCall(data);
  } else {
    const msg = JSON.parse(data.toString());
    console.log('Event:', msg);

    switch (msg.event) {
      case 'ready':
        startAudioStream(ws, phoneStream);
        break;
      case 'transfer':
        console.log('Transfer to:', msg.target);
        bridgeCallToAgent(msg.target);
        break;
      case 'voicemail_detected':
        handleVoicemail();
        break;
      case 'end':
        cleanup();
        break;
    }
  }
});

ws.on('error', (err) => {
  console.error('WebSocket error:', err);
});

ws.on('close', () => {
  console.log('Connection closed');
});

function startAudioStream(ws, phoneStream) {
  phoneStream.on('data', (chunk) => {
    ws.send(chunk, { binary: true });
  });
}

Python

Python 
import asyncio
import websockets
import json
import time

async def connect():
    uri = "wss://wellheard.ai/v1/partner/audio?api_key=YOUR_API_KEY"

    async with websockets.connect(uri) as ws:
        start_event = {
            "event": "start",
            "call_id": f"call-{int(time.time())}",
            "direction": "outbound",
            "phone_number": "+16135551234",
            "first_name": "John",
            "state": "Florida",
            "data_url": "https://your-api.com/calls/{call_id}/data",
            "webhook_url": "https://your-api.com/calls/webhook"
        }

        await ws.send(json.dumps(start_event))

        async for message in ws:
            if isinstance(message, bytes):
                await play_audio(message)
            else:
                event = json.loads(message)
                print(f"Event: {event}")

                if event["event"] == "ready":
                    asyncio.create_task(stream_audio(ws))
                elif event["event"] == "transfer":
                    bridge_call(event["target"])
                elif event["event"] == "end":
                    break

asyncio.run(connect())

Error Handling

Implement robust error handling to build a reliable integration.

Connection Errors

If your WebSocket connection is dropped, reconnect with exponential backoff.

JavaScript 
async function connectWithRetry(maxRetries = 5) {
  let retries = 0;

  while (retries < maxRetries) {
    try {
      const ws = new WebSocket(
        'wss://wellheard.ai/v1/partner/audio?api_key=YOUR_API_KEY'
      );
      return ws;
    } catch (err) {
      const backoff = Math.pow(2, retries) * 1000;
      console.log(f"Retry in ${backoff}ms");
      await new Promise(resolve =>
        setTimeout(resolve, backoff)
      );
      retries++;
    }
  }

  throw new Error('Max retries exceeded');
}

Invalid Start Event

If your start event has errors, the WebSocket connection will close with an error message. Ensure all required fields are present and correctly formatted.

Audio Timeout

If no audio is received for 30 seconds, the connection closes automatically. Keep sending audio frames at regular intervals.

Rate Limiting

Limit Value Notes
Concurrent Connections 100 per API key Maximum simultaneous WebSocket connections
Connection Duration Unlimited Individual calls can run as long as needed
Message Frequency ~50 audio frames/sec Audio must arrive at 20ms intervals
Graceful Degradation: If you exceed rate limits, new connections will be rejected. Monitor your concurrent connection count and implement a queue if needed.

Webhook Failures

WellHeard will retry failed webhooks (4xx or 5xx errors) with exponential backoff:

  • Attempt 1: Immediate
  • Attempt 2: 5 seconds
  • Attempt 3: 25 seconds
  • After 3 failed attempts, the webhook is dropped
Webhook Best Practices:
  • Return 2xx status immediately (before doing heavy processing)
  • Use idempotency keys to handle duplicate deliveries
  • Implement exponential backoff for retries
  • Log all webhook deliveries for debugging

Integration Flow

Here's the complete flow of a successful WellHeard call integration:

Step Direction Content Description
1 You → WellHeard WebSocket + JSON Open connection, send start event with call_id, customer data, and callback URLs
2 WellHeard → Your API HTTP GET Optional: WellHeard fetches lead data from your data_url
3 You → WellHeard Binary Audio Stream mulaw frames from phone system (~50/sec, 160 bytes each)
3 WellHeard → You Binary Audio AI voice audio when the AI is speaking
4 WellHeard → You JSON Events Real-time events: voicemail, transfer, end (interspersed with audio)
5 WellHeard → Your API HTTP POST Optional: Call results, transcript, and disposition
